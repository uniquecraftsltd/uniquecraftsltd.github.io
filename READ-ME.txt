  # SAPE (Smart Asset Pipeline Engine)

SAPE is a production-grade **Digital Asset Management (DAM) Core** designed to automate the lifecycle of media assets from raw capture to web-ready deployment. It transforms unstructured photo directories into a versioned, queryable, and synchronized asset repository.

## 🏗 System Architecture
SAPE follows a modular **Pipeline-Registry-Manifest** architecture, decoupling file storage from metadata management.

- **Ingestion Layer**: Recursive scanning with file-type filtering.
- **Persistence Layer**: SQLite-backed relational registry for metadata and audit trails.
- **Reconciliation Engine**: Atomic synchronization between source (RAW) and destination (WEB) using hashing to detect drift.
- **Manifest Layer**: Automated generation of `global_manifest.json` for decoupled frontend consumption.



## 🚀 Key Technical Features
- **Atomic Reconciliation**: Prevents partial synchronization errors using transactional staging.
- **Content-Addressing**: MD5-based deduplication ensures storage efficiency.
- **Many-to-Many Tagging**: Relational schema allows complex asset filtering by project, category, and tags.
- **Version Tracking**: Immutable audit trail for all asset changes.
- **CLI Interface**: Standardized CLI for pipeline operations (sync, tag, restore).

## 🛠 Tech Stack
- **Core**: Python 3.x
- **Metadata Storage**: SQLite (ACID compliant)
- **Deployment Interface**: CLI (argparse)
- **Frontend Bridge**: JSON Manifest (decoupled from backend logic)

# Unique Crafts Engineering - Asset Pipeline

## 🏗 System Architecture
This project utilizes **SAPE (Smart Asset Pipeline Engine)**, a custom-built DAM core that automates media lifecycle management.

### Key Components:
1. **Ingestion Engine**: Python-based automation that scans `raw_photos/`, handles file normalization, and mirrors the directory structure to `img/projects/`.
2. **Metadata Registry**: SQLite-backed relational storage for project tagging and audit trails.
3. **Decoupled Manifest**: Automatic generation of `global_manifest.json` for frontend consumption.
4. **Dynamic Frontend**: Isotope-based UI that dynamically injects assets without manual HTML updates.

## 🚀 How to maintain the Portfolio
- **Add New Project**: Simply drop the folder into `raw_photos/`.
- **Sync**: Run `python scripts/asset_engine.py sync`.
- **Result**: The website gallery updates automatically on next refresh.
